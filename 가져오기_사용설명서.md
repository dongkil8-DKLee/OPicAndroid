## 1. 두 가지 방법 개요

| 구분 | 방법 A: DB 백업/복원 | 방법 B: Q&A CSV |
|------|---------------------|----------------|
| 목적 | 기기 교체, 완전 백업 | 스크립트 직접 편집 |
| 파일 형식 | `.db` (바이너리) | `.csv` (텍스트) |
| 포함 데이터 | 문제·단어장·학습진도·테스트 기록 전부 | 문제 스크립트 4개 필드만 |
| 사람이 읽고 수정 가능? | ❌ (바이너리) | ✅ (Excel/구글 시트) |
| 사용자 설정 포함? | ❌ (설정은 별도 저장소) | ❌ |
| 앱 재시작 필요? | ✅ 복원 후 반드시 | ❌ |

> **원칙**: 기기 이전 → 방법 A, 스크립트 교정·작성 → 방법 B

## 2. 방법 A — DB 백업 / 복원 (전체 데이터)

### 2-1. DB 백업 절차

1. Report 화면 → **데이터 관리** 섹션 → **[DB 백업]** 버튼 클릭
2. 파일 저장 위치 선택 대화상자 → 원하는 위치에 저장
   - 기본 파일명: `shadowtalk_backup_YYYYMMDD.db`
3. "DB 백업 저장 완료" 토스트 확인

### 2-2. DB 복원 절차

1. **[DB 복원]** 버튼 클릭
2. 이전에 저장한 `.db` 파일 선택
3. "복원 완료 — 앱을 재시작해주세요" 토스트 확인
4. **앱을 완전히 종료 후 재시작** (필수)

> ⚠️ **주의**: 복원하면 현재 기기의 모든 DB 데이터가 백업 파일로 덮어씌워집니다.
> 복원 전 현재 데이터를 백업해 두세요.

### 2-3. DB에 포함되는 데이터 목록

| 테이블 | 포함 내용 |
|--------|-----------|
| Questions | 문제 텍스트, 기본 답변 스크립트, 사용자 스크립트, AI 답변 |
| User_Study_Progress | 학습 횟수, 즐겨찾기, STT 결과, 분석 결과 |
| Vocabulary | 단어, 뜻, 발음, 메모, 암기 여부, 즐겨찾기 |
| TestSession / TestResult | 테스트 세션 기록, 문제별 녹음 분석 결과 |
| ApiKey | 저장된 Claude API 키 |

### 2-4. DB에 포함되지 않는 데이터 (별도 저장소)

아래 설정들은 SharedPreferences에 저장되므로 DB 백업에 포함되지 않습니다.
기기 이전 시 앱에서 직접 재설정이 필요합니다.

| 설정 항목 | 설정 위치 |
|-----------|-----------|
| 글자 크기 | 앱 설정 |
| 테마 (라이트/다크) | 앱 설정 |
| 목표 등급 (IM1~AL) | 앱 설정 |
| TTS 음성/엔진 | 앱 설정 |
| 레벨 이미지 폴더 경로 | 앱 설정 |
| 음성 파일 폴더 경로 | 앱 설정 |
| 사용자 프로필 (직업, 취미 등) | 앱 설정 |
| Practice 화면 파형 확장 범위 | Practice 설정 |
| 문장별 타이밍 오프셋 | Practice 설정 |
| 테스트 세션 잠금 목록 | Report 화면 |

## 3. 방법 B — Q&A CSV 가져오기 (스크립트 편집)

### 3-1. 전체 흐름

```
[Q&A 내보내기] → CSV 파일 저장 → Excel/구글 시트로 편집 → [Q&A 가져오기] → DB 자동 업데이트
```

### 3-2. 내보내기 절차

1. Report 화면 → **데이터 관리** → **[Q&A 내보내기]** 버튼 클릭
2. 파일 저장 위치 선택
   - 기본 파일명: `shadowtalk_qa_YYYYMMDD.csv`
3. "Q&A CSV 저장 완료" 토스트 확인

### 3-3. CSV 파일 구조

내보내기 시 생성되는 11개 컬럼:

| 컬럼명 | 설명 | 편집 가능? |
|--------|------|-----------|
| `question_id` | 문제 고유 번호 (기본키) | ❌ 절대 수정 금지 |
| `title` | 문제 제목 | ❌ 참고용 |
| `set` | 문제 세트 | ❌ 참고용 |
| `type` | 문제 유형 | ❌ 참고용 |
| `question_text` | 질문 본문 | ✅ 편집 가능 |
| `answer_script` | 기본 답변 스크립트 | ✅ 편집 가능 |
| `user_script` | 사용자 스크립트 | ✅ 편집 가능 |
| `ai_answer` | AI 생성 답변 | ✅ 편집 가능 |
| `study_count` | 학습 횟수 | ❌ 읽기 전용 |
| `is_favorite` | 즐겨찾기 여부 | ❌ 읽기 전용 |
| `stt_grade` | STT 분석 등급 | ❌ 읽기 전용 |

### 3-4. 가져오기 절차

1. CSV 편집 완료 후 저장 (반드시 UTF-8 인코딩)
2. **[Q&A 가져오기]** 버튼 클릭
3. 편집한 CSV 파일 선택
4. "N개 문제 업데이트 완료" 토스트 확인

## 4. CSV 작성 규칙 (필수 숙지)

### 4-1. 인코딩

```
반드시 UTF-8 (BOM 없음) 로 저장
```

- Excel: 저장 시 "CSV UTF-8(쉼표로 분리)" 선택
- 구글 시트: 파일 → 다운로드 → CSV → 자동으로 UTF-8
- Notepad++: 인코딩 → "UTF-8(BOM 없음)" 선택 후 저장
- Windows 기본 메모장: ❌ ANSI 저장되므로 사용 금지

### 4-2. 헤더 행

**반드시 첫 번째 행에 컬럼명이 있어야 합니다.**

```csv
question_id,title,set,type,question_text,answer_script,user_script,ai_answer,study_count,is_favorite,stt_grade
```

- 컬럼 순서는 변경 가능 — 앱이 헤더 이름으로 자동 인식
- 불필요한 컬럼은 삭제해도 됨 (단, `question_id`는 필수)
- 컬럼명 앞뒤 공백 주의 (` question_id` ❌ → `question_id` ✅)

**최소 필수 구성 예시** (스크립트만 편집할 때):
```csv
question_id,answer_script,user_script
1,"Hello, my name is John.",""
2,"I enjoy playing tennis and reading books.","I like reading books."
```

### 4-3. 텍스트 필드 따옴표 규칙

**텍스트 값은 반드시 큰따옴표(`"`)로 감싸야 합니다.**

```csv
question_id,answer_script
1,"I am a software engineer working at a tech company."
```

**텍스트 안에 큰따옴표가 있는 경우 → 두 번 연속으로 입력**

```csv
question_id,answer_script
1,"He said ""Hello"" to me."
```

실제 저장되는 값: `He said "Hello" to me.`

### 4-4. 줄바꿈 포함 텍스트

텍스트 안에 줄바꿈(개행)이 있어도 따옴표로 감싸면 정상 인식됩니다.

```csv
question_id,answer_script
1,"First sentence.
Second sentence on a new line."
```

- 스크립트가 여러 문장으로 구성된 경우 그대로 입력
- Excel에서는 셀 안에서 Alt+Enter로 줄바꿈 입력 가능

### 4-5. 빈 값 처리

| 상황 | 입력 방법 | 결과 |
|------|-----------|------|
| 값을 비워두고 싶을 때 | `""` 또는 공백 셀 | 컬럼에 따라 다름 (아래 참조) |
| 해당 행 스킵 | question_id 없애기 | 해당 행 무시 |

### 4-6. 숫자 필드

`question_id`, `study_count`는 숫자 그대로 입력 (따옴표 없음):

```csv
question_id,study_count
1,5
2,0
```

## 5. 컬럼별 편집 규칙 상세

### 5-1. `question_id` (필수, 절대 수정 금지)

- 문제를 식별하는 **기본 키**
- 이 값을 기준으로 어떤 문제를 업데이트할지 결정
- 존재하지 않는 ID → 해당 행 완전 무시 (오류 아님)
- **절대 변경하지 마세요** — 잘못 변경하면 엉뚱한 문제가 수정됩니다

### 5-2. `question_text` (질문 본문)

- **빈 값이면 업데이트 안 함** (기존 값 유지)
- 수정하려면 반드시 내용 입력
- 삭제(빈 칸으로 만들기) 불가 — 빈 칸은 무시됨

```csv
question_id,question_text
1,"Tell me about yourself and what you do for a living."
```

### 5-3. `answer_script` (기본 답변 스크립트)

- **빈 값이면 업데이트 안 함** (기존 값 유지)
- Study 화면 [기본 답변] 탭에 표시되는 내용

```csv
question_id,answer_script
1,"I am a software engineer.
I have been working for 5 years.
I enjoy problem-solving."
```

### 5-4. `user_script` (사용자 스크립트)

- **빈 값이면 업데이트 안 함** (기존 값 유지)
- Study 화면에서 사용자가 직접 입력한 스크립트
- Practice 화면의 발화연습 STT 텍스트와 **다름** — 이 필드는 수동 입력 스크립트

```csv
question_id,user_script
1,"안녕하세요. 저는 소프트웨어 엔지니어입니다."
```

### 5-5. `ai_answer` (AI 생성 답변)

- **빈 값이면 업데이트 안 함** (기존 값 유지)
- Study 화면 [AI 답변] 탭에 표시되는 내용
- 앱에서 AI 생성한 답변을 CSV에서 직접 수정 가능

```csv
question_id,ai_answer
1,"My name is John and I work as a software engineer at a mid-sized tech company.
In my role, I primarily focus on developing mobile applications.
I have been in this field for about five years now."
```

### 5-6. 나머지 컬럼 (`title`, `set`, `type`, `study_count`, `is_favorite`, `stt_grade`)

- **가져오기 시 완전히 무시됨** — 수정해도 반영되지 않음
- 내보낸 파일을 그대로 참고용으로 보존하는 것을 권장

## 6. 편집 도구 권장 설정

### 6-1. Microsoft Excel

1. CSV 파일을 직접 열지 말고: **데이터 탭 → 텍스트/CSV에서** 사용
2. 가져오기 마법사에서 **인코딩: 65001(UTF-8)** 선택, 구분자: **쉼표**
3. 편집 완료 후 **파일 → 다른 이름으로 저장 → CSV UTF-8(쉼표로 분리)** 선택

> ⚠️ "CSV(쉼표로 분리)"와 "CSV UTF-8(쉼표로 분리)"는 다릅니다.
> 반드시 **UTF-8** 버전으로 저장하세요.

### 6-2. Google Sheets (권장)

1. Google Drive → 새로 만들기 → 파일 업로드 → CSV 선택
2. 업로드된 파일 → Google 스프레드시트로 열기
3. 편집 후: **파일 → 다운로드 → 쉼표로 구분된 값(.csv)**
4. 자동으로 UTF-8 저장됨

### 6-3. VS Code / Notepad++

- 인코딩 확인: 우측 하단에 인코딩 표시
- UTF-8 (BOM 없음)으로 저장
- 줄 끝(Line Ending): LF 또는 CRLF 모두 지원

## 7. 자주 발생하는 오류와 해결법

### 오류: "유효한 데이터 없음"

- **원인**: 파일에 헤더 행만 있거나 내용이 없음
- **해결**: 최소 2행 이상 (헤더 + 데이터 1행) 필요

### 오류: "question_id 컬럼이 없습니다"

- **원인**: 헤더 행에 `question_id` 컬럼이 없음
- **해결**: 첫 번째 행에 정확히 `question_id` 입력 (대소문자 구분)

### 오류: "파일 읽기 실패"

- **원인**: 파일이 열려 있거나 접근 권한 없음
- **해결**: Excel에서 파일을 닫은 후 다시 시도

### 오류: 한글이 깨져서 들어옴 (모자이크 처럼 표시)

- **원인**: UTF-8이 아닌 인코딩으로 저장
- **해결**: [6. 편집 도구 권장 설정] 참조, UTF-8로 다시 저장 후 재가져오기

### 오류: 줄바꿈이 없어지고 한 줄로 합쳐짐

- **원인**: Excel에서 셀 내 줄바꿈이 제거됨
- **해결**: Excel에서 편집 시 셀을 더블클릭 후 Alt+Enter로 줄바꿈 입력

### 오류: 업데이트 개수가 예상보다 적음

- **원인**: `question_id`가 DB에 없는 값이거나 숫자가 아님
- **해결**: 내보내기 한 원본 CSV의 `question_id` 값을 그대로 유지했는지 확인

## 8. 어떤 데이터가 DB에 포함되는가

### DB 백업/복원 포함 데이터 (완전 복원 가능)

```
opic.db
├── Questions 테이블
│   ├── question_text     (질문 본문)
│   ├── answer_script     (기본 답변 스크립트)
│   ├── user_script       (사용자 스크립트)
│   └── ai_answer         (AI 생성 답변)
│
├── User_Study_Progress 테이블
│   ├── study_count       (학습 횟수)
│   ├── is_favorite       (즐겨찾기 여부)
│   ├── stt_text          (마지막 STT 인식 텍스트)
│   └── analysis_result   (발음 분석 결과 JSON)
│
├── Vocabulary 테이블
│   ├── word              (단어)
│   ├── meaning           (뜻)
│   ├── pronunciation     (발음)
│   ├── memo              (메모)
│   ├── isMemorized       (암기 여부)
│   └── isFavorite        (즐겨찾기)
│
├── TestSession 테이블   (테스트 세션 기록)
├── TestResult 테이블    (세션별 문제 결과)
└── ApiKey 테이블        (Claude API 키)
```

### DB 백업/복원 미포함 (앱 재설정 필요)

```
SharedPreferences (앱 설정 파일)
├── opic_app_settings
│   ├── 글자 크기
│   ├── 테마 (라이트/다크)
│   ├── 목표 등급
│   ├── TTS 음성 / 엔진
│   ├── 레벨 이미지 폴더
│   ├── 음성 파일 폴더
│   └── 사용자 프로필 (직업/취미/가족/국가/배경)
│
├── practice_settings
│   ├── 파형 확장 범위 (앞/뒤 ms)
│   └── 문장별 타이밍 오프셋
│
└── opic_session_locks
    └── 잠긴 테스트 세션 목록
```
